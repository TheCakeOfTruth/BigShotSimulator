Le tableau bidimensionnel duquel on est parlé est dans «scripts/attacks/RollerCoaster.as»
Il y a aussi plusieurs Dictionary dans plusieurs scripts, qui pourraient qualifier comme un «tableau associatif,» ainsi que plusieurs Objects de notations {}.
Dans la section des ITEMS du menu, tu peux importer et exporter des données en forme de XML. Ces données sont vérifiées pour assurer que les valeurs nécessaires sont présentes.
Pour le «fichier externe qui contient les données initiales du programme [incorporé] à mon code principal,» je dirais que les fichiers dans le dossier «lang» fonctionneraient. Les langages sont en forme de «.AS» au lieu de «.XML» pour améliorer la fonctionnalité de l’«include» dans LocalizationHandler.
Il y a probablement plusieurs instances qui pourraient qualifier comme des algorithmes de recherche, mais il y a quelques-uns auquel je peux penser dans «scripts/Bullet.as».

Il y a quelques petites parties de code que j’ai trouvé en ligne, mais ils sont plutôt mineurs et servent seulement des fonctions de débogage (voir Main.as), ou ils sont des équations mathématiques (voir rotateToSmart() dans scripts/spam/SpamtonPart.as) simples. Des informations sur les statistiques du jeu original (Kris, Spamton, les items) sont dérivées soit de mes observations personnelles pendant que je jouais le jeu (https://store.steampowered.com/app/1671210/DELTARUNE/), ou sur un page dans le wiki du jeu (https://deltarune.fandom.com/wiki/Main_Page) (il y a quelques liens dans les scripts).

Les sons ont été extraits du jeu, j’avais besoin de convertir des sons «.OGG» en «.WAV» ou «.MP3» pour que Flash puisse les utiliser.
Les images proviennent du SpritersResource (soit https://www.spriters-resource.com/pc_computer/deltarune/ ou https://www.spriters-resource.com/pc_computer/undertale/)

Il fallait que je crée des symboles de texte avec des fonts téléchargés, car je n’ai pas pu réussir à utiliser ces fonts avec le code ActionScript, malheureusement.

Il y a quelques petits morceaux de code dans le scénario, tu peux les tous trouver facilement en jouant le jeu avec le «debug mode» de Flash (CTRL+SHIFT+ENTER, au lieu de CTRL+ENTER). Debug Mode c’est aussi la seule manière d’être capable de jouer le jeu dans l’éditeur de Flash, car ça va désactiver les raccourcis du clavier pour les outils de l’éditeur. Autrement, les raccourcis de Flash sont prioritaires sur les KeyboardEvents que les contrôles du jeu utilisent, et tu ne serais pas capable de contrôler le jeu hors du menu.

Si tu as de la difficulté à comprendre le jeu, lis les instructions dans le menu. Si tu as de la difficulté à survivre, essaye d’utiliser un des préréglages que j’ai faits en «préréglages d’items,» ils ont le meilleur équipement, et les deux ensembles représentent tous les items que j’ai programmés.
